# Phase 8: ポストリリース機能改善とUI強化

リリース後のフィードバックに基づき、ユーザビリティの向上とワークフローを効率化する機能を追加します。

## 1. 文字起こし機能の強化と編集
会話内容の活用を促進するための機能群です。

- [ ] **全会話セグメントの一括文字起こしと保存**
  - メニューに「Transcribe All Conversations」を追加。
  - バックグラウンドですべての会話セグメントを順次文字起こしする。
  - 完了後、テキストファイル (.txt) として保存するダイアログを表示、またはクリップボードにコピーする機能。
- [x] **文字起こしテキストの編集機能**
  - Inspectorの文字起こし結果表示エリアを `TextEditor` (編集可能) に変更。
  - 編集内容を `AudioSegment` モデルに反映し、プロジェクト保存時に保持されるようにする。
- [x] **文字起こし表示エリアのUI改善**
  - 現在の固定幅/狭い表示領域を見直し、ウィンドウ幅に応じて最大限広がるようにレイアウトを調整。
  - 必要に応じてInspectorの幅を可変にするか、別ウィンドウ/別パネルでの表示を検討（後述のSplit View調整と関連）。

## 2. プロジェクト管理の効率化
ファイルを開く際の手間を軽減し、継続的な作業をサポートします。

- [ ] **プロジェクトファイルの自動検出と読み込み確認**
  - 音声ファイル読み込み時、同一ディレクトリに同名のプロジェクトファイル (`{filename}.rehearsallink.json`) が存在するかチェック。
  - 存在する場合、アラートを表示し「プロジェクトとして読み込む」か「音声のみ読み込む（新規）」かを選択可能にする。

## 3. 編集機能の拡充
セグメント編集の柔軟性を高めます。

- [ ] **セグメント結合機能 (Merge)**
  - 分割 (Split) の対となる機能。
  - 隣接する2つ以上のセグメントを選択し、1つのセグメントに統合する。
  - 統合時のタイプ判定（左側優先など）や、文字起こしテキストの結合（改行で繋ぐなど）を考慮。

## 4. UIレイアウトの柔軟性向上
- [x] **波形・Inspector領域の可変区切り (Adjustable Split View)**
  - メイン画面を `HSplitView` (SwiftUIでは `HSplitView` または `GeometryReader` + DragGestureによるカスタム実装) に変更。
  - ユーザーが波形エリアとInspectorエリアの比率をドラッグで調整できるようにする。

## 5. ドキュメントの最新化
- [ ] **README.md の大幅刷新**
  - アーキテクチャ解説中心の内容から、ユーザーベネフィット（このアプリで何ができるのか、どんな課題が解決されるのか）を強調した構成に変更。
  - リハーサル録音の聞き返し効率化、文字起こしによる内容把握など、進化した現状の機能を強力にアピール。

## 6. CI/CD 環境の整備
- [ ] **Release Build 用の GitHub Workflow 作成**
  - タグ打ちなどのトリガーで、macOS アプリのビルド、バイナリのアーカイブ（ZIP作成）、GitHub Release への自動アップロードを行うワークフローの実装。
